# Improvement Plan — 14-mechanical-checks

**Skill:** review
**Change type:** New capability (broadens Quality + Security modes)

## Problem

`/review` currently covers semantic and test-execution layers but skips the mechanical layer: lint, format, typecheck, and dedicated secret scanning. The semantic Quality mode catches design-level issues but cannot flag an unused import or a type error; Coverage runs tests but not `tsc --noEmit`; Security's "code-review for hardcoded creds" is explicitly *not* a scanner — [SKILL.md:31](../../../skills/review/SKILL.md#L31) refuses dedicated secret scanning. The result: PRs reach `gh pr create` with mechanical defects (broken types, leaked tokens, lint failures) that `/review` silently passed. Merging these checks into existing modes (rather than adding new ones) keeps the five-mode pipeline intact while giving `/review` the same floor a local pre-commit hook would provide.

## Decisions captured (from intake)

- **Verdict mapping:** lint + format = `warn`; typecheck + secrets = `block`.
- **Run ordering in Quality:** mechanical gate runs *before* `/cook` semantic agents. Any `block` short-circuits the mode and skips `/cook` entirely (fast-fail, saves tokens on broken code).
- **Missing tool:** if a runner is configured in the repo but not executable (e.g. `eslint` in `package.json` with no `node_modules/`), emit `block` with a clear remediation note. Never silently skip — environment drift hides real failures.
- **Secret-scan scope:** diff-only by default; `/review --full-secret-scan` opts into full-tree scan.
- **Runner coverage:** JS/TS (`eslint`, `prettier`, `tsc`), Python (`ruff`, `black`, `mypy`), Dart/Flutter (`dart analyze`, `dart format`), secret scanners (`gitleaks` preferred, `trufflehog` fallback). Detection layer designed runner-agnostically — domain → runner map lives in `FLAG-LIST.md` so adding Rust/Go later is a table edit.

## Out of scope

- **Git pre-commit hook wiring** — explicitly decided in intake: all checks run inside `/review` on-demand. No `.husky/` or `.pre-commit-config.yaml` is created by this plan.
- **Auto-fix mode** — `/review` does not modify source ([SKILL.md:31](../../../skills/review/SKILL.md#L31) refusal stands). Lint `--fix` and prettier `--write` are out; we only report.
- **New mode** — no new mode is added. The merge is into existing Quality + Security modes only. Coverage is untouched.
- **Auto-install on missing runner** — explicitly decided as `block`, not auto-`npm install`. Environment fixes are the user's job.
